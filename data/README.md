# Локальні дані

GitHub-репозиторій не містить ринкових даних, кешів або підготовлених
наборів.

Під час першого `quarto render` проєкт:

1. отримує Binance, Bybit і Bitstamp за однаковий період;
2. перевіряє джерело, параметри, OHLC, часові межі та пропуски;
3. створює RDS-кеші в `data/cache/`;
4. створює набір ціни й дохідностей у `data/processed/`;
5. записує SHA-256 у локальний `data-manifest.yml`.

Усі створені файли ігноруються Git. Якщо вони вже існують і
відповідають `config.yml`, повторний рендер не завантажує їх знову.

Binance використовується лише для початкового порівняння повноти з
Bybit. Підготовлений набір ціни й дохідностей створюється тільки з
основного ряду Bybit. Bitstamp зберігається окремо як контроль.
