# Встановлення оновлення етапу 2

## Що змінюється

1. Прибирається ручна нумерація з підзаголовків.
2. Усі графіки отримують однакову висоту 860 px.
3. Графіки займають ширшу область сторінки.
4. Годинний графік показує 90 днів.
5. Чотиригодинний графік показує один рік.
6. Додається швидке Parquet-сховище Binance.
7. Додаються інтервали `5m`, `15m`, `30m`, `1h`, `2h`, `4h`, `6h`, `12h`, `1d`.
8. Додається вибіркове читання хвилинних даних через DuckDB.
9. Додаються контрольні ряди Bybit `4h` і `1d`.
10. До книги додається порівняльний графік Binance і Bybit.
11. Додається денна таблиця для моделювання без витоку майбутніх даних.
12. Додаються словник колонок, часові вибірки та метадані витрат.

## Необхідні пакети

У консолі RStudio:

```r
renv::install(c(
  "data.table",
  "DBI",
  "duckdb"
))
```

Після встановлення перезапустіть R-сесію.

## Перший запуск швидкого сховища

```r
source(
  "scripts/03_update_binance_fast.R",
  encoding = "UTF-8"
)
```

Перший запуск завантажує і перевіряє потрібні ZIP-архіви, а потім перетворює їх на Parquet. Він може тривати довше за наступні запуски.

source(

## Побудова окремих HTML-графіків

```r
source(
  "scripts/02_validate_and_build_charts.R",
  encoding = "UTF-8"
)
```

Цей скрипт використовує те саме швидке сховище і не завантажує весь хвилинний ряд у пам'ять.

## Дані для моделювання

```r
source(
  "scripts/05_build_model_data.R",
  encoding = "UTF-8"
)
```

Скрипт створює денну таблицю, у якій ознаки відомі до моменту рішення, а ціль належить наступному періоду. Результати, словник колонок, часовий поділ і метадані витрат зберігаються у `data/model/binance/spot/BTCUSDT/`.
