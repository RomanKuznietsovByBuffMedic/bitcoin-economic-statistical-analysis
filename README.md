# Bitcoin і BTC: економічний та статистичний аналіз

[Читати онлайн-книгу](https://romankuznietsovbybuffmedic.github.io/bitcoin-economic-statistical-analysis/)

## Головне питання

> Чи може статистика допомогти отримувати прибуток від торгівлі BTC?

Проєкт не виходить із припущення,
що статистична модель обов'язково дасть змогу заробляти.

Мета - перевірити це на реальних даних
з урахуванням правил торгівлі, витрат і ризиків.

## Для кого цей проєкт?

Для звичайної людини,
яка не знає економіки Bitcoin і складних статистичних моделей.

Спочатку все пояснюється простими словами.
Математика, статистика й програмний код додаються поступово,
коли вже зрозуміло, навіщо вони потрібні.

## Як побудоване дослідження?

1. Зрозуміти, що таке Bitcoin і BTC.
2. З'ясувати, чому BTC має ринкову ціну.
3. Отримати й перевірити реальні дані.
4. Сформулювати конкретне статистичне питання.
5. Вибрати найпростішу придатну модель.
6. Створити правило купівлі, продажу або бездіяльності.
7. Перевірити результат на нових даних після врахування витрат.
8. Ускладнювати модель лише тоді, коли це справді потрібно.

> Модель сама не заробляє гроші.
> Перевіряти потрібно модель разом із правилом дій,
> витратами та ризиком.

## Практичні межі

- перший ринок: Binance Spot `BTCUSDT`;
- торгівля без кредитного плеча;
- орієнтація на невеликий бюджет;
- перевірка результатів на даних,
  яких модель не бачила під час побудови.

## Що вже реалізовано?

- створено відтворюваний проєкт на R, Quarto та `renv`;
- опубліковано онлайн-книгу через GitHub Pages;
- підготовлено початкові розділи про Bitcoin, BTC і ринкову ціну;
- створено отримання та перевірку даних Binance Spot `BTCUSDT`;
- додано інтерактивні графіки ціни, дохідності, обсягу та волатильності.

## Поточний етап

Наступне завдання - сформулювати першу вузьку гіпотезу
та побудувати мінімальний практичний експеримент.

Конкретна модель обиратиметься лише після того,
як буде зрозуміло, яке питання вона має перевірити.

## Відтворення

Відновити пакети R:

```r
renv::restore()
```

Побудувати HTML-книгу:

```bash
quarto render
```

Результат буде створено в каталозі `_book/`.
